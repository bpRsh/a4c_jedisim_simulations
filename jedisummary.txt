1. Jedicatalog
* Info        : This program takes in 302 galaxy fitsfile from input folder and
 *               reads some parameters from these fitsfiles.
 *               (simdatabse/bulge_disk_f8/f8_bulge_disk0_to_301.fits)
 *
 *               It also reads other parameters from other input text files:
 *                   1. config.conf
 *                   2. lens.txt
 *                   3. psf.txt
 *                   5. simdatabase/radius_db/20_29.dat     # r50 radius database
 *                   6. simdatabase/red_db/19_99.dat        # redshift database
 *
 *               Then, in the end it creates three catalog list textfiles:
 *                   1. config_output_folder/trial0_catalog.txt
 *                   2. config_output_folder/trial0_convolvedlist.txt
 *                   3. config_output_folder/trial0_distortedlist.txt
 *
 *              And, these catalog list will be read by jedicolor and jeditransform programs.
 
 
 2. Jeditransform
 
 * This program takes in the catalog list created by jedicatalog:
 * (e.g. out1/trial0_catalog.txt)
 *
 * reads the galaxies names which are to be transformed
 * (e.g. simdatabase/bulge_disk_f8/f8_bulge_disk0.fits
 *       
 * and also read other parameters needed to transform that galaxy from fitsfiles headers
 * (e.g. input_name x y angle redshift pixscale old_mag old_r50 new_mag new_r50 stamp1 stmap2)
 *
 * Note: example of a row of input catalog_list
 *
 * image    = simdatabase/bulge_disk_f8/f8_bulge_disk271.fits
 * x        = 5571.742676
 * y        = 6911.588867
 * angle    = 254.850693
 * redshift = 1.500000
 * pixscale = 0.030000
 * old_mag  = 23.273899
 * old_r50  = 0.326880
 * new_mag  = 26.170000
 * new_r50  = 0.131700
 * stamp1   = out1/stamp_0/stamp_0.fits.gz
 * stamp2   = out1/distorted_0/distorted_0.fits
 *
 *
 * Then, it creates 12420 zipped fitsfiles inside
 * out1/stamp_0/stamp_0_to_999.fits.gz  (for stamps 0 to 12 )
 *
 * It also creates dislist.txt for the jedidistortion,viz.,
 * out1/trial0_dislist.txt
 * 
 * Row of distlist.txt from jeditransform for jedidistort:
 * 5838    1741   13       7        1.500000  out1/stamp_0/stamp_0.fits.gz  out1/distorted_0/distorted_0.fits
 * xembed  yembed x-width  y-width  redshift  out_stamp_from_jeditransform  fitsfile_from_jedidistort   
 * 
 *   x - x coord. of lower left pixel where galaxy should be embedded
 *   y - y coord. of lower left pixel where galaxy should be embedded
 *   nx - width of the galaxy in pixels
 *   ny - height of the galaxy in pixels
 *   zs - redshift of this galaxy
 *   infile - filepath to the FITS file for this galaxy, 1024 chars max
 *   outfile - filepath for the output FITS file for this galaxy, 1024 chars max
 
 3. Jedidistort
 
 * Info        : This program distorts the 12420 galaxies from out1/stamp_0_to_stamp_12/fitsfiles
 *               according to dislist.txt and lens.txt and write distorted
 *               galaxies inside out1/distorted_/
 
 4.  * Info: This program combines 12,420 distorted fits files inside the out1/distorted_/distorted_fits/
 *       into a single large embedded image: out1/trial0_HST.fits.
 *
 *       It reads the left corners of images (nx,ny) from the config file: config.conf
 
 5. Jediconvolve
  * Info: This program will convolve the HST.fits (i.e. out1/trial0_HST.fits)
 *       with the given psf (i.e. psf/psf0.fits) and writes 6 bands of convolved images
 *       (i.e. out1/convolved/convolved_band_0_to_5.fits )
 
 6. Jedirescale
 
 * Info: This program scales down convolved HST image to LSST image.
 * i.e. out1/trial0_HST_convolved.fits to out1/rescaled_convolved_lsst_0.fits
 *
 
 7. Jedinoise
  * Info        : This program adds Poisson noise to a given input fitsfile.
 *               e.g. with exposure time 6000 seconds and noise mean 10,
 *               we can add noise to fitsfile "trial0_LSST_convolved.fits"
 *               to get "trial0_LSST_convolved_noise.fits"
